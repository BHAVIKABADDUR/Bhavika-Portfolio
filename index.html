Frame 03

What I've built

Four projects, roughly the order I'd bring them up in an interview. FinSight is the one I'd want you to open first.

01 flagship

FinSight Enterprise AI

Multi-agent financial intelligence platform

This one started with a design question I couldn't let go of: what if AI agents were never allowed to touch the database directly? FinSight reads bank statements and invoices, pulls structured data out of them, and routes the review work through five LangGraph agents, but every one of those agents talks to the data through four MCP tool servers I built specifically to sit in between. It's overkill for a side project, and I built it that way on purpose. The current dataset covers 387+ transactions across multiple business entities, with Power BI handling the reporting layer on top.

Built with: LangGraph · MCP · n8n · Supabase · Qdrant · Groq (Llama 3.3) · Power BI · PySpark · Tesseract OCR

02

DataForge AI

Multi-agent analytics copilot

This came out of RetailIQ, honestly. I got tired of hopping between SQL, notebooks, and dashboards just to answer one question about the data. So I built a seven-agent setup that figures out on its own whether a request needs SQL, analysis, documentation, or an ETL job, then goes and does it. It writes queries and PySpark code, explains what it found, and holds onto enough context to handle a follow-up question without me repeating myself.

Built with: LangGraph · LangChain · Groq · FAISS · SQL Server

03

RetailIQ

Retail intelligence lakehouse

Before this, I thought data cleaning basically was the pipeline. RetailIQ is where that stopped being true: a Bronze, Silver, Gold lakehouse in Databricks for retail transactions across four regions, feeding into SQL Server and then Power BI. The part that actually stuck with me: a clean dataset is worthless if the person after you can't query it without asking what half the columns mean.

Built with: Databricks · PySpark · Delta Lake · SQL Server · Power BI

Source
04

Hinglish Sentiment Analysis

Code-mixed NLP for e-commerce reviews

I built this because sentiment models fall apart the second a review switches from Hindi to English mid-sentence, which is just... how people actually write. I tried classical ML against XLM-RoBERTa and ended up collapsing it to a binary sentiment setup, which got me to 77% accuracy. The score mattered less than what it showed me: how fast a model trained on clean English text stops working the moment real language shows up.

Built with: Python · XLM-RoBERTa · scikit-learn · Streamlit
