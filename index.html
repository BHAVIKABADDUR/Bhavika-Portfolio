Frame 03

What I've built

These projects are in roughly the order I would talk about them in an interview. Some are more complete than others, but each one pushed me into a part of engineering I had not worked with before.

01 flagship

FinSight Enterprise AI

Multi-agent financial intelligence platform

FinSight started with a question I kept coming back to: what would an AI workflow for financial operations look like if I treated access and control as part of the design, not something added later? The system reads bank statements and invoices, extracts structured data, and routes review work through five LangGraph agents. I built four MCP tool servers between the agents and the database so the agents do not query it directly. The current dataset covers 387+ transactions across multiple business entities, with Power BI used for the reporting layer.

Built with: LangGraph · MCP · n8n · Supabase · Qdrant · Groq (Llama 3.3) · Power BI · PySpark · Tesseract OCR

02

DataForge AI

Multi-agent analytics copilot

DataForge came from working on RetailIQ and getting tired of moving between SQL, notebooks, and dashboards for every question. I built a seven-agent workflow that decides whether a request needs SQL, analytics, documentation context, or ETL help. It can generate queries and PySpark code, explain results, and keep enough conversation context to understand follow-up questions.

Built with: LangGraph · LangChain · Groq · FAISS · SQL Server

Source
03

RetailIQ

Retail intelligence lakehouse

RetailIQ was where I stopped treating data cleaning as the whole pipeline. I built a Bronze, Silver, and Gold lakehouse in Databricks for retail transactions across four regions, then modelled the reporting data in SQL Server and connected it to Power BI. The project taught me why a clean dataset is only useful if the next person can actually query and understand it.

Built with: Databricks · PySpark · Delta Lake · SQL Server · Power BI

Source
04

Hinglish Sentiment Analysis

Code-mixed NLP for e-commerce reviews

I built this after noticing how awkward sentiment models become when a review switches between Hindi and English in the same sentence. I compared classical machine learning approaches with XLM-RoBERTa and eventually used a binary sentiment setup that reached 77% accuracy. More than the score, the project taught me how quickly real language breaks assumptions made on clean English datasets.

Built with: Python · XLM-RoBERTa · scikit-learn · Streamlit

Source
05

Resume Matching System

NLP-based HR tooling

This was an early NLP project built around a simple problem: keyword matching misses resumes that describe the right experience using different words. I combined TF-IDF with semantic similarity and tested the matcher on more than 300 resumes. In my evaluation, it improved shortlisting precision by 15% compared with plain keyword matching.

Built with: Python · TF-IDF · NLP · FastAPI

Source
06

UniEats

Smart cafeteria platform

UniEats started much closer to home. I spoke to more than 60 students about the college cafeteria and kept hearing the same complaints about queues and not knowing what was available. I built a small ordering platform around pre-orders, queue tracking, and dietary filters. It was one of my first projects where talking to users changed what I built.
